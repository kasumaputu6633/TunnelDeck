The config values below are identical across OSes; only the install + activate commands differ.

1. Install WireGuard on the node

Debian / Ubuntu:

sudo apt update
sudo apt install -y wireguard wireguard-tools

Arch / CachyOS:

sudo pacman -S --needed wireguard-tools

Fedora / RHEL:

sudo dnf install -y wireguard-tools

2. Generate a keypair on the node

wg genkey | sudo tee /etc/wireguard/node.key | wg pubkey | sudo tee /etc/wireguard/node.pub
sudo chmod 600 /etc/wireguard/node.key

Copy the value printed from node.pub — you'll paste it into step 4.

3. Write the WireGuard config on the node

Replace <PASTE_NODE_PRIVATE_KEY> with the contents of /etc/wireguard/node.key.

sudo tee /etc/wireguard/wg0.conf > /dev/null <<'EOF'
{{.Data.Config}}EOF
sudo chmod 600 /etc/wireguard/wg0.conf

5. Bring up the tunnel on the node

sudo systemctl enable --now wg-quick@wg0

sudo wg show

1. Install WireGuard on the node

The easiest path is the WireGuard app from the Mac App Store — it gives you a menubar icon and handles activation on boot:

https://apps.apple.com/app/wireguard/id1451685025

Or via Homebrew for the CLI tools:

brew install wireguard-tools

2. Generate a keypair on the node

In the WireGuard app: Add Empty Tunnel (⌘N). A new keypair is generated automatically; copy the Public key field — you'll paste it into step 4.

CLI alternative:

wg genkey | tee ~/node.key | wg pubkey > ~/node.pub
chmod 600 ~/node.key
cat ~/node.pub   # paste this into step 4

3. Paste the WireGuard config

In the WireGuard app's tunnel editor, replace the contents with:

{{.Data.Config}}

The PrivateKey line is pre-filled by the app when you used Add Empty Tunnel; leave it alone and only change Address, [Peer], Endpoint, AllowedIPs, and PersistentKeepalive.

5. Activate the tunnel on the node

Toggle the switch next to the tunnel name in the WireGuard app, then verify:

ping -c3 {{stripCIDR .Data.Gateway.WGIP}}

Enable On-Demand in the tunnel settings to have the Mac reconnect automatically.

1. Install WireGuard on the node

Download and install the official Windows client:

https://www.wireguard.com/install/

After install, open the WireGuard application from the Start menu.

2. Create a tunnel (auto-generates a keypair)

In the WireGuard UI, click Add Tunnel → Add empty tunnel… (Ctrl+N). The app generates a new private/public keypair and shows them in the editor. Copy the Public key — you'll paste it into step 4.

3. Paste the WireGuard config

Replace the config text in the editor with the block below. Keep the PrivateKey = ... line that the app already filled in; only change the other fields to match:

{{.Data.Config}}

Give the tunnel a name (e.g. TunnelDeck) and click Save.

5. Activate the tunnel on the node

Click the tunnel in the list and press Activate. Verify from an Administrator PowerShell:

ping {{stripCIDR .Data.Gateway.WGIP}}

Right-click the WireGuard tray icon → Settings → Start automatically if you want it to reconnect at boot.

4. Register the node's public key on the gateway

Run this on the gateway (this VPS). Replace <NODE_PUBLIC_KEY> with the value you copied in step 2:

# add the peer at runtime (takes effect immediately)
sudo wg set {{.Data.Gateway.WGIf}} peer <NODE_PUBLIC_KEY> allowed-ips {{.Data.Node.WGIP}}/32 persistent-keepalive {{.Data.Node.Keepalive}}

# persist to disk so the peer comes back on reboot
sudo tee -a /etc/wireguard/wg0.conf > /dev/null <<'EOF'

[Peer]
# tunneldeck:node={{.Data.Node.Name}}
PublicKey = <NODE_PUBLIC_KEY>
AllowedIPs = {{.Data.Node.WGIP}}/32
PersistentKeepalive = {{.Data.Node.Keepalive}}
EOF

Step 4 is the same regardless of which OS the node runs — the gateway doesn't care.
