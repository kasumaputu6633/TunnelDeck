Setup: {{.Data.Node.Name}}

Assigned WG IP: {{.Data.Node.WGIP}}

1. Install WireGuard on the node

Ubuntu/Debian:

sudo apt update
sudo apt install -y wireguard wireguard-tools

Arch/CachyOS:

sudo pacman -S --needed wireguard-tools

2. Generate a keypair on the node

Run this on the node (the private key must never leave the node):

wg genkey | sudo tee /etc/wireguard/node.key | wg pubkey | sudo tee /etc/wireguard/node.pub
sudo chmod 600 /etc/wireguard/node.key

Copy the node.pub value — you'll paste it into the gateway's wg0.conf in step 4.

3. Write the WireGuard config on the node

Replace <PASTE_NODE_PRIVATE_KEY> with the contents of /etc/wireguard/node.key. 

sudo tee /etc/wireguard/wg0.conf > /dev/null <<'EOF'
{{.Data.Config}}EOF
sudo chmod 600 /etc/wireguard/wg0.conf
{{if not .Data.Gateway.WGPublicKey}}
Gateway public key hasn't been detected yet. Once TunnelDeck reads it from wg show, the config above will update automatically.
{{end}} {{if hasPrefix .Data.Endpoint "<"}}
Gateway public IP isn't set. Fill it in on the Settings page so the node knows where to dial.
{{end}}

4. Add the node's public key to the gateway

On the gateway (this VPS), add the node as a peer to the running WireGuard interface and to wg0.conf so it survives reboots. Replace <NODE_PUBLIC_KEY> with the contents of /etc/wireguard/node.pub from the node. 

# add the peer at runtime (takes effect immediately)
sudo wg set {{.Data.Gateway.WGIf}} peer <NODE_PUBLIC_KEY> allowed-ips {{.Data.Node.WGIP}}/32 persistent-keepalive {{.Data.Node.Keepalive}}

# persist to disk so the peer comes back on reboot
sudo tee -a /etc/wireguard/wg0.conf > /dev/null <<'EOF'

[Peer]
# tunneldeck:node={{.Data.Node.Name}}
PublicKey = <NODE_PUBLIC_KEY>
AllowedIPs = {{.Data.Node.WGIP}}/32
PersistentKeepalive = {{.Data.Node.Keepalive}}
EOF

(A future release will do this for you via a one-time join token. For now it's a manual step because the node hasn't sent its public key yet.)

5. Bring up the tunnel on the node

sudo systemctl enable --now wg-quick@wg0

# verify:
sudo wg show
ping -c3 {{stripCIDR .Data.Gateway.WGIP}}

6. Verify from TunnelDeck

Go back to the Nodes page and refresh. The node should show online with a recent handshake. Once you see that, add port forwards for it on the Forwards page.
